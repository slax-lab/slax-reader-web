## 三、bookmark fixture 建设（第三期 Sprint 1 前置）

第二期 PLAN §5.2 第七轮记录：`bookmark.ts` fixture 推迟到第三期，届时拉本地后端反推字段。

### 3.1 数据来源

- 本地后端：`http://localhost:8787`（需启动 `slax-reader-api-community`）
- 测试 token：见会话历史（不写入代码）
- 接口：`GET /v1/bookmark/:id`（需 Authorization header）

### 3.2 需要的 fixture 类型

```ts
// tests/fixtures/bookmark.ts
import type { BookmarkDetail, BookmarkBriefDetail, InlineBookmarkDetail, ShareBookmarkDetail } from '@commons/types/interface'

export const baseBookmarkDetail: BookmarkDetail = {
  /* 从真实接口反推 */
}
export const makeBookmarkDetail = (overrides: Partial<BookmarkDetail> = {}): BookmarkDetail => ({ ...baseBookmarkDetail, ...overrides })

export const baseBookmarkBrief: BookmarkBriefDetail = {
  /* ... */
}
export const makeBookmarkBrief = (overrides: Partial<BookmarkBriefDetail> = {}): BookmarkBriefDetail => ({ ...baseBookmarkBrief, ...overrides })

// 同款：ShareBookmarkDetail / InlineBookmarkDetail
```

### 3.3 字段注意事项

- `bookmark_id?: number`（optional）
- `archived: 'inbox' | 'archive' | 'later'`（字符串字面量，不是 boolean）
- `starred: 'star' | 'unstar'`（字符串字面量）
- `status: BookmarkParseStatus`（enum）
- ID 长度：通常 19 位数字（从真实接口确认）

## 四、e2e 基础设施建设（第三期 Sprint 2）

### 4.1 技术选型

**Playwright**（推荐）：

- 支持 Chromium / Firefox / WebKit
- 内置 `page.waitForSelector` / `page.click` / `page.fill` 等
- 与 Vitest 并行运行（不冲突）
- 支持 `test.use({ storageState })` 注入登录态

### 4.2 目录结构

```
apps/slax-reader-dweb/
├── tests/
│   ├── e2e/                    ← 新建
│   │   ├── fixtures/
│   │   │   └── auth.ts         ← 登录态 fixture（storageState）
│   │   ├── pages/
│   │   │   ├── bookmarks.spec.ts
│   │   │   └── article.spec.ts
│   │   └── utils/
│   │       └── pwa.spec.ts     ← pwa.ts 第三期补
│   └── ...（现有 unit/integration 不动）
├── playwright.config.ts        ← 新建
└── package.json                ← 加 test:e2e script
```

### 4.3 playwright.config.ts 基础配置

```ts
import { defineConfig, devices } from '@playwright/test'

export default defineConfig({
  testDir: './tests/e2e',
  fullyParallel: true,
  forbidOnly: !!process.env.CI,
  retries: process.env.CI ? 2 : 0,
  workers: process.env.CI ? 1 : undefined,
  reporter: 'html',
  use: {
    baseURL: 'http://localhost:3000',
    trace: 'on-first-retry'
  },
  projects: [{ name: 'chromium', use: { ...devices['Desktop Chrome'] } }],
  webServer: {
    command: 'pnpm dev:dweb',
    url: 'http://localhost:3000',
    reuseExistingServer: !process.env.CI
  }
})
```

### 4.4 登录态 fixture

```ts
// tests/e2e/fixtures/auth.ts
import { test as base } from '@playwright/test'

export const test = base.extend({
  page: async ({ page }, use) => {
    // 注入 token cookie（从环境变量读，不硬编码）
    await page.context().addCookies([
      {
        name: process.env.COOKIE_TOKEN_NAME || 'token',
        value: process.env.TEST_TOKEN || '',
        domain: 'localhost',
        path: '/'
      }
    ])
    await use(page)
  }
})
```

## 七、第三期技术风险

| 风险 | 概率 | 缓解 |
| --- | --- | --- |
| useBookmark.ts 依赖 useScroll / useScrollLock / useDebounceFn（vueuse）在 happy-dom 下行为不稳定 | 中 | vi.stubGlobal 替换 window.scrollY / vi.useFakeTimers 控制 debounce |
| Playwright e2e 与 pnpm dev:dweb 启动时间长，CI 超时 | 中 | webServer.reuseExistingServer=true 本地复用；CI 单独 job + 超时 5min |
| ChatBot.vue 965 行，流式渲染 + SSE 在 happy-dom 下不稳定 | 高 | 优先 e2e（真实浏览器），unit test 只测 ChatBot class（已完成） |
| bookmark fixture 字段与真实接口不一致（后端改动） | 低 | fixture 从真实接口反推 + 定期 `curl` 验证 |
| useNotification 的 navigator.serviceWorker.ready 在 happy-dom 下不可用 | 高 | vi.stubGlobal('navigator', { serviceWorker: { ready: Promise.resolve({ ... }) } }) |

## 八、第三期工具约定

1. **继续使用 codex-review-loop.sh**：每个 sprint spec 文档落盘后过 codex review，直到通过
